APP SCREENSHOT SPECIFICATIONS

Each screenshot should have a clean device frame (no bezels), with the app content centered.
Use a subtle green gradient background (#0E6B3A to #07592F) on the outside of the device frame.
Text overlay: Minimal — feature title in English (top) and Arabic (bottom) in white, using a clean sans-serif font.

DEVICE FRAMES (3 required):

1. iPhone 6.5" Display (iPhone 14 Pro Max / 15 Plus)
   - Resolution: 1290 x 2796 px (portrait)
   - File naming: screenshot_en_1.png through screenshot_en_5.png
   - Arabic variants: screenshot_ar_1.png through screenshot_ar_5.png

2. iPad 12.9" Display (iPad Pro 12.9")
   - Resolution: 2048 x 2732 px (portrait)
   - File naming: screenshot_ipad_en_1.png, screenshot_ipad_en_2.png
   - Arabic: screenshot_ipad_ar_1.png, screenshot_ipad_ar_2.png

3. Android 6.2" Display (Google Pixel 7 / similar)
   - Resolution: 1080 x 2400 px (portrait)
   - File naming: screenshot_android_en_1.png through screenshot_android_en_4.png
   - Arabic: screenshot_android_ar_1.png through screenshot_android_ar_4.png

DESIGN GUIDELINES:
- Use the app's actual UI content in the screenshots
